# .planning

Project planning & knowledge. Markdown only — no code, no generated output.

## Layout

| File / Dir | Purpose |
|------------|---------|
| `ROADMAP.md` | Phases, milestones, what's next |
| `todos/` | Task lists — one file per workstream or topic |

## Conventions

- One concern per file. Link between files with relative paths.
- Date entries you'll revisit: `YYYY-MM-DD`.
- Keep it skimmable — bullets and tables over prose.
